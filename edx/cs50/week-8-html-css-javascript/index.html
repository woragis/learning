Protocols

IP internet protocol

Wires connecting every possibility of network? No

That's why we use routers

each router pass the connection search to another router, until you find what you are searching

Ip gets information from point A to point B

Ip brakes the data into small "packets"

TCP Transfer Control Protocol

It has total control over the packets that are being transferred through the routers

That's why the real protocol is called TCP/IP because the transfer occurs through ip protocol, but with the help of TCP

TCP can deal with data loss along the way without needing to request the full data again

HTTP Hypertext Transfer Protocol

Basically is the formatting of use of the other protocols

Successful interaction

Request

        GET / HTTP/1.1
        Host: cats.com
        ...

Response

        HTTP/1.1 200 OK
        Content-type: text/html
        ...

Fail interaction

Request

        GET / HTTP/1.1
        Host: cats.com
        ...

Response

        HTTP/1.1 404 Not Found
        Content-type: text/html
        ...

Request Structure

Method - request-target - Http version

Response Structure

Http version - Http status
Class Code Text Comments
Success 200 OK All is well, valid request and response
Redirection 301 Moved Permanently Page is now at a new location, automatic redirects built in to most browsers
Redirection 302 Found Page is now at a new location temporarily
Client Error 401 Unauthorized Page typically requires login credentials
Client Error 403 Forbidden Server will not allow this request
Client Error 404 Not Found Server cannot find what was asked for
Server Error 500 Internal Server Error Generic server failure in responding to the otherwise-valid request

Other Protocols

  • FTP File transfer protocol uses port 21
  • SMTP email transfer protocol uses port 25
  • DNS uses port 53
  • HTTP Web Browsing uses port 80
  • HTTPS secure web Browsing uses port 443
